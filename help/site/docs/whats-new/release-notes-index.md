# Release notes index

This section is the product-facing changelog and guidance area for Procedo Help. It is where users should look when they want to know:

- what shipped recently
- whether a change is additive or breaking
- what the current support model is
- which limitations are intentional in the current phase
- what the likely next areas of investment are

## How this section should evolve

As Procedo grows, each release should add:

1. a dedicated release-notes page
2. upgrade notes when behavior or syntax changes
3. links to affected reference and how-to pages
4. a short compatibility statement

The goal is to make release notes useful to working users, not just to record internal implementation history.
