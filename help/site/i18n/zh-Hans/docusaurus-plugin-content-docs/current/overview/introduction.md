Procedo 是一个基于 .NET 的工作流引擎，用 YAML 定义流水线，支持依赖感知执行、插件化步骤、持久化、回调式恢复以及可观测性。

它适合这样的场景：你希望工作流可以被审查、可重复执行，并且比一组零散脚本更容易维护和运维。

这个帮助站点优先强调实用性：

- 简短直接的说明
- 经过验证的示例
- 面向任务的操作指南
- 精确的参考文档

## 你可以用 Procedo 做什么

- 使用 `stages`、`jobs` 和 `steps` 用 YAML 定义工作流。
- 通过输出和表达式在步骤之间传递数据。
- 使用运行时 `condition:` 控制是否执行某个步骤。
- 使用模板以及模板阶段的分支/迭代来复用工作流结构。
- 对长时间运行或等待审批的流程进行持久化和恢复，包括宿主侧的回调式恢复。
- 通过插件和自定义步骤扩展引擎能力。

## 这套文档适合谁

- 编写 YAML 工作流的用户
- 将 Procedo 嵌入 .NET 应用的工程师
- 负责运行和检查工作流的运维或操作人员
- 扩展运行时能力的插件作者

## 这套文档有什么不同

这个站点的目标是成为真正的用户参考文档，而不仅仅是项目介绍。

这意味着：

- 示例应该可以实际运行
- 命令应该针对当前仓库状态完成验证
- 大主题应该拆分成更小、更容易搜索的页面
- 文档不仅列字段，还要解释行为和使用方式
