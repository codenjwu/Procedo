# 发布说明索引

这个部分是 Procedo Help 面向用户的变更说明与指引区域。当用户想了解以下内容时，应该先看这里：

- 最近发布了什么
- 某个变化是增量变化还是破坏性变化
- 当前支持模型是什么
- 当前阶段有哪些限制是刻意保留的
- 接下来最可能投入的方向有哪些

## 这个部分后续应该如何演进

随着 Procedo 持续发展，每个版本都应该补充：

1. a dedicated release-notes page
2. upgrade notes when behavior or syntax changes
3. links to affected reference and how-to pages
4. a short compatibility statement

目标是让发布说明对真实使用者有帮助，而不仅仅是记录内部实现历史。
